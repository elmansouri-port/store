Design Principles

The four values that govern every decision — from micro-interactions to page layout.

01 — Clarity over decoration
Every visual element must earn its place. Glassmorphism and gradients create depth, not noise. If removing something makes the page cleaner, remove it.
02 — Motion with purpose
Animation guides attention, confirms state, and signals hierarchy. Never animate for entertainment. Spring easing cubic-bezier(.16,1,.3,1) is our signature curve.
03 — Trust through consistency
Enterprise buyers make decisions on perceived reliability. Consistent spacing, type, and color patterns signal craftsmanship. Inconsistency erodes confidence.
04 — Accessible by default
Contrast ratios, focus states, and keyboard navigation are non-negotiable. AA compliance is the floor, not the ceiling.

Issues to Fix

Design debt found during the audit, prioritized by user impact.

P0
Dual brand purple. #5e2d91 vs #7c3aed are used interchangeably as brand.500. Standardize in Tailwind config across ALL pages: brand.500 = #5e2d91, add brand.vibrant = #7c3aed for glass borders and glow effects.
P0
Two FAQ implementations. .faq-item (style.css, older) and .faq-card (style.css, Laurenn style) coexist. Migrate all pages to .faq-card and remove .faq-item.
P1
Two button systems. .btn-cta-primary (border-radius 12px, shadow) and Tailwind inline buttons (rounded-btn, 10px) are both used. Prefer .btn-cta-primary everywhere except in-nav buttons.
P1
Border radius inconsistency. Tailwind classes (rounded-xl = 12px) mixed with raw CSS (border-radius: 20px, 24px). Create a shared token map or consolidate custom values into Tailwind config.
P1
No CSS custom properties. All tokens are scattered across inline Tailwind config per-page and raw CSS. Extract into a :root {} token block in style.css to enable single-source theming.
P2
SVG icons are hardcoded inline. No central icon system. Consider an SVG sprite or a small icon component map to enable consistent sizing and color via stroke.
P2
Sparse sm: breakpoints. Most responsive jumps go mobile → md:/lg:, skipping the 640px range. This creates jarring transitions on tablet. Add sm: steps to key grid layouts.

Do / Don't

Core principles in practice.

✓ Do
Use spring easing for all hover transitions
cubic-bezier(.16,1,.3,1) is the brand signature. Use it on every lift, expand, and slide.
✗ Don't
Mix ease-in-out and linear for UI transitions
Reserve linear for ambient animations only (marquee, glow pulse). Never for interactive states.
✓ Do
Use #5e2d91 for all primary brand moments
Logo area, primary CTA, word animation, section badges. This is the ALE brand color.
✗ Don't
Use #7c3aed as the primary brand color
This vibrant purple is for glass borders, glow effects, and interactive accent states — not brand identity.
✓ Do
Use letter-spacing: -0.03em on all headings
Tight tracking at large sizes is a design hallmark that makes headings feel premium and intentional.
✗ Don't
Add decorative elements without purpose
Blobs, gradients, and shine effects exist to create depth. If they don't serve the content, remove them.
✓ Do
Tint shadows with brand purple
rgba(124,58,237,…) not rgba(0,0,0,…) for elevated components. Keeps the palette cohesive.
✗ Don't
Use pure black shadows on colored surfaces
Black shadows on purple/indigo backgrounds look muddy. Purple-tinted shadows keep cards feeling part of the system.
